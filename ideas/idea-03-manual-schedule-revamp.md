# 手動班表改版

## 功能需求

### 1. 跨日多時段框選
目前拖曳框選限制在單日欄位內，改版後要與時段安排表（`/settings/demand`）一致，支援跨多日、多小時的連續框選。

**跨日衝突語意**：若框選範圍內有些格子已有排班、有些格子為空——
- 全部已有排班 → 清除模式
- 任意一格為空 → 指派模式

### 2. 互動模型：先框選 → 再操作
框選放開後，於左側員工清單或工具列執行動作：

- **指派**：框選完成後，在左側員工清單點選員工 → 立即填入選取範圍
- **清除**：框選完成後，點工具列的「清除」按鈕 → 行內二次確認後批次刪除

（原本的 dropdown 員工選單移除，改由左側清單承擔選人角色。）

### 3. 左側員工清單：可用性標記 + 搜尋過濾
框選有效範圍時，清單中每位員工顯示對該範圍的可用性：

| 標記 | 說明 |
|------|------|
| ✓ 全段可用 | 選取範圍內所有格子皆標記為可排班 |
| ⚠ 部分可用 | 部分格子未標記 |
| ✗ 未標記 | 範圍內無任何可用格子 |

搜尋／過濾功能可等員工清單頁面（Phase 2）完成後復用 API，不需提前實作。

### 4. Undo（移除）
改版後不實作 undo。操作足夠明確（二次確認清除），不需要額外的撤銷機制。

## 不採納的方案

- **時段安排表加入「無法排班」第三狀態**：灰色（未選）已足以表達不可排班，加入第三態會使員工填寫介面複雜化（3 態 toggle），且語意與現有灰色重疊。可用性判斷改由手動排班側邊欄計算呈現（見第 3 點）。
